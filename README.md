# Cursor LATAM Kit 🌎

**El kit en español para llevar Cursor a desarrolladores y equipos de Latinoamérica.**
*The Spanish-language kit for bringing Cursor to LATAM developers and teams.*

> Recursos abiertos para que cualquier persona en LATAM pase de "instalé Cursor" a "mi equipo entrega más rápido con IA" — sin que el idioma sea la barrera.

---

## ¿Por qué existe esto? / Why this exists

La mayoría del contenido de calidad sobre Cursor está en inglés. En LATAM hay una comunidad enorme de devs y equipos de producto que adoptarían Cursor más rápido si tuvieran:

1. **Guías en español** pensadas para sus stacks reales (no traducciones literales).
2. **Flujos de trabajo** que conecten Cursor con prácticas ágiles (Scrum/Kanban), no solo "autocompletar".
3. **Una forma lista para usar de organizar eventos** locales (meetups, hackathons, cafés Cursor).

Este repo cubre los tres. *Most quality Cursor content is in English. This kit closes that gap for the LATAM community across three fronts: Spanish-first guides, agile-aware workflows, and a ready-to-run community playbook.*

## El marco que uso: Planear → Delegar → Revisar

Vengo de mundo ágil (Scrum/Kanban), así que enseño Cursor como un **miembro junior muy rápido de tu equipo**, no como una varita mágica.

- **Planear** — defines la historia/tarea y los criterios de aceptación *antes* de pedirle código (igual que un buen refinamiento de backlog).
- **Delegar** — le das a Cursor contexto suficiente (`rules`, archivos, objetivo) y dejas que haga el trabajo pesado.
- **Revisar** — tú sigues siendo el responsable del *merge*. La IA acelera, pero el criterio es humano.

> Analogía: Cursor es como contratar a alguien brillante en su primer día. Si le das una tarea vaga, entrega algo vago. Si le das una historia bien escrita y contexto, vuela. El playbook enseña a escribir esas "historias" para la IA.
